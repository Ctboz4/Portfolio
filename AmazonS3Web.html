I created and deployed a static website using Amazon S3, AWS's cloud storage service, to host HTML content and supporting assets. The project involved setting up an S3 bucket, configuring static website hosting, managing file permissions, and troubleshooting access issues.

What I learned

  • Gained hands-on experience with Amazon S3 for website hosting
  • Worked with HTML files and structured supporting assets to create a fully functional static website.
  • Familiarity with AWS S3 configuration, bucket policies, and access control lists (ACLs).
  • Resolved issues related to file permissions and access errors (403 Forbidden) effectively.

Steps I took

Set Up the S3 Bucket

  • Created an S3 bucket in the Ohio region to store website files.
  • Ensured a globally unique bucket name as required by S3.

Uploaded Files

  • Uploaded the main index.html file for website structure.
  • Added a supporting folder with images and other assets for styling and functionality.

Enabled Static Website Hosting

  • Configured the bucket to act as a static website by enabling hosting settings
  • Obtained the bucket's endpoint URL to access the hosted website.

Troubleshooting

  • Diagnosed and resolved file access issues by updating permissions and re-uploading corrected files.

Challenges I faced

  • Encountered a "403 Forbidden" error when accessing the website initially due to private file settings. Resolved by adjusting permissions and making the necessary files public
  • Learned the importance of organizing and maintaining the correct folder structure for website assets.

What I Learned

  • Gained an understanding of how Amazon S3 stores and serves data for website hosting.
  • Learned how to configure permissions and troubleshoot common access errors.
  • Realized the significance of precise file and folder organization in cloud-based projects.
  • Enhanced my ability to identify and resolve technical issues efficiently.

Outcome

The static website was successfully deployed and accessible via the S3 bucket's endpoint URL. This project strengthened my understanding of AWS services and demonstrated my ability to apply cloud-based solutions effectively.
